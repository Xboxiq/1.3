# مهارة: تصميم UI/UX احترافي

## متى تُستخدم
عند تصميم واجهة جديدة، أو وضع نظام تصميم، أو مراجعة جودة شاشة، أو اتخاذ قرارات لون/خط/تخطيط/حركة.

## المبادئ السبعة (طبّقها في كل شاشة)
1. **التباين (Contrast):** نص عادي ≥ 4.5:1 وكبير ≥ 3:1 (WCAG 2.2). لا تعتمد على اللون وحده لنقل المعنى.
2. **الطباعة (Typography):** مقياس خط واضح (مثل 12/14/16/20/24/32)، ارتفاع سطر مريح (1.5)، حد أقصى لطول السطر ~70 محرفاً.
3. **التركيز (Focus):** حالة `:focus-visible` ظاهرة دائماً، ترتيب tab منطقي، لا تُزِل الـ outline دون بديل واضح.
4. **الحركة (Motion):** انتقالات هادفة 150–300ms، واحترم `prefers-reduced-motion: reduce`.
5. **مرونة الإدخال:** دعم الماوس + اللمس + لوحة المفاتيح؛ أهداف لمس ≥ 44×44px.
6. **وضوح المحتوى:** لغة بسيطة، عناوين واضحة، حالات فارغة/تحميل/خطأ مكتوبة بعناية.
7. **استرجاع الأخطاء:** رسائل خطأ محددة قابلة للتنفيذ، منع الخطأ قبل وقوعه، تأكيد للإجراءات الخطيرة.

## نظام التصميم أولاً (Design Tokens)
عرّف رموزاً (لا قيماً مبعثرة) لـ: الألوان (مع حالات hover/active/disabled)، المسافات (مقياس 4/8)، الزوايا، الظلال، الخطوط، نقاط الكسر (breakpoints). اربط الوصولية بالرموز نفسها بحيث يكون الخيار الصحيح هو الافتراضي.

## Microinteractions
لكل تفاعل: مُحفّز (Trigger) → استجابة فورية → تغذية راجعة لحالة النظام. هدف واحد لكل تفاعل، خفيفة، تحترم تقليل الحركة، وتنقل الحالة (نجاح/خطأ/جارٍ).

## الإجراء (Procedure)
1. حدّد المستخدم والمهمة الأساسية و"رحلة" الإنجاز.
2. ضع نظام التصميم/الرموز قبل بناء المكوّنات.
3. صمّم الحالات كلها: افتراضي/تحميل/فارغ/خطأ/نجاح/معطّل.
4. تحقق من التباين والتركيز والتنقل بلوحة المفاتيح.
5. أضِف microinteractions هادفة فقط.
6. راجع التجاوب على موبايل/تابلت/سطح المكتب.

## فحوص القبول (Checklist) ✅
- [ ] رموز تصميم موحّدة، لا قيم سحرية مبعثرة.
- [ ] تباين ألوان مطابق لـ WCAG 2.2 AA.
- [ ] تركيز ظاهر + تنقل كامل بلوحة المفاتيح + أدوار/تسميات ARIA صحيحة.
- [ ] كل الحالات مصممة (فارغ/تحميل/خطأ/نجاح).
- [ ] الحركة تحترم `prefers-reduced-motion`.
- [ ] تجاوب سليم على كل المقاسات.

> ملاحظة RTL: للعربية اعتمد منطق start/end لا left/right (انظر مهارة web-development).
